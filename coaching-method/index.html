CORE PROTOCOL

認知が本当に動く、
問いの順番。

出来事の意味づけや支援へ早く進むのではなく、まず感情を生んでいる片側認知を見つけます。プラス側ならデメリット、マイナス側ならメリットを一問だけ置き、緩みが起きた後に初めて支援を扱います。

01 / EVENT

出来事

何が起きたのか。解釈を混ぜず、具体的な事実から始める。

02 / EMOTION

感情

怒り、悔しさ、悲しみ、恥、虚しさ。不快感の種類を言葉にする。

03 / FRAMEWORK

片側認知

感情を生んでいる「〇〇は良い / 〇〇はダメ」という前提を見つける。

04 / DIRECTION

向きを判定

プラス側か、マイナス側か。ここで問いの向きを決める。

05 / PLUS

プラス側なら

「〇〇であることのデメリットは何ですか？」と問う。

06 / MINUS

マイナス側なら

「〇〇であることのメリットは何ですか？」と問う。

07 / RELEASE

緩みを確認

片側だけの見方が緩んだか、本人の感覚とことばで確認する。

08 / INTEGRATION

支援を見る

緩んだ後で初めて、出来事に含まれる支援や次の実践を扱う。

PLUS SIDE

プラス側フレームワーク

例: 「誤解なく伝わることはいいこと」「共感されることはいいこと」

「共感されることのデメリットは何ですか？」
MINUS SIDE

マイナス側フレームワーク

例: 「誤解されることはダメ」「否定されることはダメ」

「誤解されることのメリットは何ですか？」

片側認知が緩む前に、「何を受け取ったか」「何を失ったか」「どんな支援があるか」へ進みません。それは思考や前向きな解釈へ早く進ませる可能性があるためです。

FIELD NOTES

実践から生まれた
コーチング法則

成果に意識が向いた時、人は支援を見失う。

望んだ結果だけを価値と見ていると、過程で受け取ったものが見えなくなる。

支援を見る前に、前提を見る。

前提を見つけたら、プラス側にはデメリット、マイナス側にはメリットを短く問う。

感情は、認知の入口である。

感情が、どの片側認知を探すべきかの入口になる。

構造が見えた瞬間ほど、感情に戻る。

コーチの仮説ではなく、本人の内側から構造が立ち上がることを大切にする。

「もうやらない」は、回避のサインであることがある。

統合は、同じ出来事が起きても違う見方ができる状態として現れる。

クライアントの言葉を保持すると、問いの精度が上がる。

言葉の定義を本人に確認することで、コーチ側の意味づけによるずれを小さくする。

深く見る前に、受け取る。

感情と言葉を一度受け止めてから問うことで、本人が自分の足で進める場になる。

見えた構造は、短い一問に変換する。

コーチの解説より、本人の口から出た言葉が次の選択に残る。

PRACTICE SCIENCE

法則は、実践で
検証し続ける。

ここで扱う「科学」は、個人の思想を正解として掲げることではありません。複数のセッション記録から観察可能なサインを抽出し、同じ問いを試し、本人の言葉・感情・次の選択に何が起きたかを記録して更新する実践体系です。

観察できるサインがあるか: 感情、身体反応、本人の中心語、発話の変化。
再現できる問いにできるか: 短い一問、反対側を見る問い、統合確認。
本人の反応で確認できるか: 感情が変わったか、選択に余白が生まれたか。
反例を残せるか: 支援を急いだ時、問いが長い時、機能しなかった条件も記録する。
COACH'S CHECK

コーチ自身が
見落とさないために。

出来事からすぐ構造へ進まず、クライアントの感情を聞いたか。
質問を一度に重ねず、一つの問いとして渡せているか。
具体の出来事と、抽象化した前提を混同せず確認できているか。
自分の見立てに答えてもらう時間ではなく、本人の言葉が増える場になっているか。
